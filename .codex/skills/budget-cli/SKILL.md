# budget-cli

## Purpose
Python CSV-based budget CLI app work for this repository.

## When to use
- Implementing CLI budget features.
- Writing or reviewing TDD tests for CSV transaction handling.
- Checking complexity, typing, and maintainability for this project.

## Working rules
- Write tests before implementation.
- Keep functions typed and short.
- Prefer simple data structures and explicit transaction fields.
- Validate changes with `pytest` and `radon cc`.

## Review checklist
- Types present on every function.
- Functions stay under 50 lines.
- Complexity stays at or below 10.
- Edge cases are covered by tests.
- QA review happens before commit.
