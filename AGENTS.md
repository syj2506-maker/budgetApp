# AGENTS.md

## 프로젝트 설명
이 저장소는 CSV 기반 가계부 CLI 앱 프로젝트다.
사용자는 CSV 파일에 저장된 거래 데이터를 읽고, 가계부 관련 기능을 CLI에서 확인하고 실행한다.

## 코딩 규칙
- 모든 함수와 메서드에는 타입 힌트를 반드시 작성한다.
- 함수 하나의 길이는 최대 50줄을 넘기지 않는다.

## TDD 규칙
- 구현보다 테스트를 먼저 작성한다.
- 새 기능은 실패하는 테스트를 먼저 추가한 뒤 구현한다.

## 품질 규칙
- 순환복잡도는 10 이하로 유지한다.
- 복잡도가 높아질 경우 함수 분리나 조기 반환으로 단순화한다.

## 품질 검수 규칙
- 커밋 전에 반드시 `qa_engineer` 서브에이전트로 품질 검수를 수행한다.
- 타입 힌트, 테스트 누락, 순환복잡도, 엣지 케이스를 함께 점검한다.

## 테스트 실행 명령어
- `pytest`
- `radon cc`

## 커밋 규칙
- 하나의 기능이 개발되면 즉시 커밋한다.
- 필요한 경우 커밋 후 푸시까지 진행한다.
